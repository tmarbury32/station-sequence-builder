    Scope notes: generates a ladder logic (RLL) routine using a DINT step sequencer (EQU/OTE/MOV pattern), plus controller tags for every input and output you define, with descriptions carried over. Discrete/analog I/O default to plain Base tags with documentation-only physical addresses — map those to real I/O modules in Studio 5000 after import. EtherNet/IP devices flagged "Uses EDS" instead generate Alias tags pointing at that module's real members (ModuleTagName:I/O.Member), matching how Studio 5000 actually exposes EDS-based device data. Those aliases only resolve once you've separately installed the device's EDS file (Tools → EDS Hardware Installation Tool) and added it as a Module in the I/O tree under the matching Module Tag Name — that step happens in Studio 5000 itself, before or after importing this routine. Member names/types you enter here should be verified against the Module Definition dialog or the real EDS, since format varies by vendor. In Studio 5000: right-click the target Program → Import Routine, or right-click Controller Tags → Import Tags, and select the generated .L5X file. Review tag data types, radixes, and any "create tag" prompts before downloading to your controller.
